Important Information

Deploying a new pool

  • Name: The name of the pool to be displayed in the Fuse directory. Private pools are not displayed in the public directory.
  • Close Factor: The percent, ranging from 0% to 100%, of a liquidatable account's borrow that can be repaid in a single liquidate transaction. If a user has multiple borrowed assets, the closeFactor applies to any single borrowed asset, not the aggregated value of a user‚Äôs outstanding borrowing. Compound's liquidation incentive is 0.5.
  • Max Assets: The maximum number of assets a single account can enter/participate in (borrow or use as collateral).
  • Liquidation Incentive: The additional collateral given to liquidators as an incentive to perform liquidation of underwater accounts. For example, if the liquidation incentive is 1.1, liquidators receive an extra 10% of the borrowers collateral for every unit they close. Compound's liquidation incentive is 1.08.
  • Price Oracle: Secure and reliable price oracles are a top priority when it comes to lending--that's why we have such a diverse range of price oracles available. While the ChainlinkPriceOracle is the most secure, it only supports 36 assets. On the other hand, the UniswapView and UniswapAnchoredView oracles support virtually any combination of assets, and the PreferredPriceOracle combines the ChainlinkPriceOracle and the UniswapView to maximize the range of supported assets while maximizing the security of larger assets. Just beware of price manipulators in Uniswap markets with low liquidity. Read on below for more information and warnings.
  • Privacy: Public pools are listed in the public Fuse pool directory, while private pools are hidden. Sometime in the future, we will add a whitelisting feature for private pools to prevent external use.

Deploying a new asset

  • Interest Rate Model: The interest rate model chosen for an asset defines the rates of interest for borrowers and suppliers at different utilization levels. See below for an explanation of each of our available interest rate models.
  • Name: The currency symbol for the interest-bearing ERC20 token created on top of the underlying asset. Defaults to the pool name + the underlying currency symbol (i.e., Fuse ETH, Fuse DAI, Fuse USDC).
  • Symbol: The currency symbol for the interest-bearing ERC20 token created on top of the underlying asset. Defaults to "f" (for Fuse) + the underlying currency symbol (i.e., fETH, fDAI, fUSDC).
  • Decimals: The number of decimal places for the cToken. Defaults to 8.
  • Collateral Factor: A cToken's collateral factor can range from 0-90%, and represents the proportionate increase in liquidity (borrow limit) that an account receives by minting the cToken. Generally, large or liquid assets have high collateral factors, while small or illiquid assets have low collateral factors. If an asset has a 0% collateral factor, it can't be used as collateral (or seized in liquidation), though it can still be borrowed. On Compound, a healthy collateral factor is 0.6 for most currencies and 0.75 for ETH, USDC, DAI, and other highly liquid currencies. However, we will note that Compound's USDT collateral factor is 0, possible due to the controversry surrounding Tether.
  • Reserve Factor: The fraction of interest generated on a given asset that is routed to the asset's Reserve Pool. The Reserve Pool protects lenders against borrower default and liquidation malfunction. For example, a 5% Reserve Factor means that 5% of the interest that borrowers pay for that asset would be routed to the Reserve Pool instead of to lenders. Thus, as a market grows, it is important that the Reserve Pool grow commensurately in order to adequately protect lenders. Generally, more volatile assets should have higher Reserve Factors. On Compound, while more volatile assets use 0.25, ETH uses 0.2, DAI uses 0.15, and USDC and USDT use 0.075.
  • Admin Fee: The fraction of interest generated on a given asset that is routed to the asset's admin address as a fee.

Price oracles

The following price oracle contracts are arranged in order of most popular to least popular. We recommend that any Fuse pool that can use the ChainlinkPriceOracle for all assets should do so. If Chainlink does not supports all assets in your Fuse pool, we recommend using the public PreferredPriceOracle, which is a combination of ChainlinkPriceOracle and UniswapView (but please read the details and warnings about UniswapView below to best ensure oracle integrity). For those who are running private Fuse pools or for those using trusted price reporters, UniswapAnchoredView allows you to source prices from your own reporter, anchored by Uniswap prices (again, please read the details and warnings about UniswapView below to best ensure oracle integrity).

‚úîÔ∏è ChainlinkPriceOracle

The ChainlinkPriceOracle provides secure and reliable on-chain price data for the following 36 currencies: AAVE, ANT, BAL, BNT, BUSD, BZRX, COMP, CRO, DAI, DMG, ENJ, ETH, KNC, LINK, LRC, MANA, MKR, MLN, NMR, REN, REP, RLC, SNZ, sUSD, TUSD, UMA, UNI, USDC, USDT, WBTC, WETH, WNXM, WOM, YFI, and ZRX.

‚ö†Ô∏è PreferredPriceOracle (ChainlinkPriceOracle + UniswapView)

The PreferredPriceOracle prioritizes prices available via ChainlinkPriceOracle and uses UniswapView for all other prices. See above and below for more information on these underlying price oracles.

  • ‚ö†Ô∏è Public PreferredPriceOracle: See "Public UniswapView" below.
  • ‚ö†Ô∏è‚ö†Ô∏è Private PreferredPriceOracle: See "Private UniswapView" below.
‚ö†Ô∏è UniswapView (Uniswap TWAPs)

The UniswapView price oracle provides on-chain price data via TWAPs (time-weighted average prices) of Uniswap V2 pairs based in ETH. If you are deploying your own UniswapView or using prices for which others are not posting TWAPs, you will need to run a bot to compute TWAPs and post them to the UniswapView as described here. Note that insufficient Uniswap market liquidity and/or an insufficient period of time over which the TWAP is computed may put it at risk of manipulation. Also note that the more often you compute and post TWAPs and the more assets you do so for, the more ETH you will spend on gas fees. However, also note that infrequent updates to an asset's price could leave room for attackers to profit via arbitrage at the expense of your users.

  • ‚ö†Ô∏è Public UniswapView: Anyone can spend the gas necessary to regularly refresh TWAPs for any asset with an ETH-based Uniswap V2 pair. All token configurations are verified by the contract for correctness. There is no reason not to use the public UniswapView unless you need to use SushiSwap (or another Uniswap V2 fork) or you need to set fixed prices for tokens other than WETH.
  • ‚ö†Ô∏è‚ö†Ô∏è Private UniswapView: Anyone can spend the gas necessary to regularly refresh TWAPs for any asset with an ETH-based Uniswap V2 pair, SushiSwap pair, or any other Uniswap V2 pair fork. However, token configurations added by the UniswapView administrator are not checked for correctness, though token configurations cannot be modified once they are added. This means that if you are using a private UniswapView that is not yours, you should check the token configurations you plan to use before adding those assets to your Fuse pool.
‚ö†Ô∏è UniswapAnchoredView (Coinbase Pro or another reporter anchored by Uniswap TWAPs)

The UniswapAnchoredView is the same price oracle used by Compound Finance. UniswapAnchoredView outputs verified prices signed by Coinbase Pro and posted to the Open Price Feed and anchors these prices via TWAPs of Uniswap V2 pairs for extra security. You can also use your own custom reporter instead of Coinbase Pro. (While the default reporter is Coinbase Pro, those prices are available without you having to spend gas via the ChainlinkPriceOracle.)

  • ‚ö†Ô∏è Secure prices for 12 currencies via Coinbase Pro: Compound ensures that Coinbase Pro prices are regularly and reliably posted for the following 10 currencies: BAT, COMP, DAI, ETH, KNC, LINK, REP, UNI, WBTC, and ZRX. Furthermore, USDC and USDT can be pegged to the value of 1 USD using the ETH/USD price. While you will not need to post prices from Coinbase Pro, until we build a proxy for Compound's UniswapAnchoredView, you will need to update prices by regularly running postPrices(bytes[] calldata messages, bytes[] calldata signatures, string[] calldata symbols) where messages and signatures are empty arrays and symbols always includes ETH.
  • ‚ö†Ô∏è All prices signed by Coinbase Pro: Anyone, including Fuse pool administrators, can spend the gas necessary to post any price signed by Coinbase Pro. To post prices, run postPrices(bytes[] calldata messages, bytes[] calldata signatures, string[] calldata symbols) using data from here. While you will not need to post ETH prices from Coinbase Pro, until we build a proxy for Compound's UniswapAnchoredView, you will need to update prices regularly via postPrices, though you do not need to include a message or signature for ETH. Note that the more often you post prices and the more assets you do so for, the more ETH you will spend on gas fees. However, also note that infrequent updates to an asset's price could leave room for attackers to profit via arbitrage at the expense of your users.
  • ‚ö†Ô∏è‚ö†Ô∏è Reporters other than Coinbase Pro: A private UniswapAnchoredView is used to post prices signed by a reporter other than Coinbase Pro. To post prices, run postPrices(bytes[] calldata messages, bytes[] calldata signatures, string[] calldata symbols). Please note that the security of a private UniswapAnchoredView mostly depends on the security of your private reporter. Note that since prices are posted in USD and outputted in ETH, you will need to post ETH prices even if ETH is not an asset in your Fuse pool. Note that the more often you post prices and the more assets you do so for, the more ETH you will spend on gas fees. However, also note that infrequent updates to an asset's price could leave room for attackers to profit via arbitrage at the expense of your users.

Interest rate models

‚úîÔ∏è WhitePaperInterestRateModel

This parameterized model is described in section 2.4 of the original Compound Protocol whitepaper. The contract accepts a base rate and slope parameter in its constructor.

‚úîÔ∏è JumpRateModel

The JumpRateModel is the standard interest rate model used by Fuse pools (as well as Compound Finance). The contract accepts a base rate, base slope, jump slope (kicks in after the kink), and kink (utilization rate above which the jump slope kicks in).

Fuse's Safe Liquidator

When running a Fuse pool, you need a bot to liquidate unhealthy loans. Fortunately, Fuse's Safe Liquidator contract allows liquidators to safely liquidate loans on any Fuse Pool by confirming on-chain that the liquidator will not lose money on each liquidation, so you will likely have external liquidators working for you.
